# GitShop 购物流程

本文档详细描述了 GitShop 平台的完整购物流程，从浏览商品到完成支付的全过程。

## 购物流程概述

GitShop 的购物流程基于 GitHub 平台的各种功能，实现了一个无服务器的电子商务解决方案。整个流程包括：商品浏览、购物车管理、订单提交、支付处理和订单确认等环节。

## 详细流程

### 1. 浏览商品

1. 用户访问基于 GitHub Pages 托管的 GitShop 前端页面
2. 浏览商品列表，可按类别筛选或搜索特定商品
3. 查看商品详情，包括价格、描述、图片等信息
4. 查看用户评价和评分（如果有）

### 2. 购物车管理

1. 用户选择心仪的商品，点击"加入购物车"按钮
2. 系统将商品添加到购物车，并显示购物车图标上的商品数量
3. 用户可以继续购物或进入购物车页面
4. 在购物车页面，用户可以：
   - 调整商品数量
   - 移除不需要的商品
   - 查看商品小计和总计金额
   - 查看适用的折扣和优惠

### 3. 结算过程

1. 用户在购物车页面点击"结算"按钮
2. 如果购物车中包含实物商品，系统会要求用户填写收货信息：
   - 收件人姓名
   - 联系电话
   - 详细地址
   - 备注信息（可选）
3. 系统使用 RSA 公钥加密用户的敏感信息
4. 系统生成订单摘要，包括：
   - 订单号
   - 商品列表及数量
   - 价格小计
   - 折扣金额
   - 最终支付金额

### 4. 订单提交

1. 系统将加密后的用户信息和订单详情整合为 GitHub Issue 内容
2. 用户点击"提交订单"按钮，系统引导用户创建一个新的 GitHub Issue
3. Issue 标题格式为：`Order: [订单号]`
4. Issue 内容包括：
   - 订单摘要（明文）
   - 加密的用户信息（如收货地址）
   - 订单创建时间
   - 其他必要信息

### 5. 支付处理

1. 提交 Issue 后，GitHub Actions 自动触发后端处理流程
2. 后端使用私钥解密用户信息
3. 系统生成唯一的 USDT 支付地址
4. GitHub Actions 自动回复 Issue，提供：
   - USDT 支付地址
   - 支付金额
   - 支付二维码（如适用）
   - 支付说明

### 6. 支付确认

1. 用户使用任何支持 USDT 的钱包应用向指定地址转账
2. 后端监控系统自动检测支付状态
3. 支付确认后，GitHub Actions 自动更新 Issue 状态：
   - 更改 Issue 标签从 `pending-payment` 到 `paid`
   - 在 Issue 中添加支付确认信息
   - 通过 Telegram Bot 通知商家有新的已支付订单

### 7. 订单处理

1. 对于数字商品：
   - 系统自动在 Issue 中回复下载链接或激活码
   - 更新 Issue 标签为 `completed`
   
2. 对于实物商品：
   - 商家收到通知后处理订单
   - 商家在发货后更新 Issue，提供物流信息
   - 更新 Issue 标签为 `shipped`

### 8. 订单完成

1. 用户收到商品或服务后，可以在 Issue 中留下反馈
2. 商家可以回复用户的问题或感谢用户的购买
3. 当订单完全处理完毕后，Issue 可以被关闭，标记为 `closed`

## 分销流程

如果用户是通过分销链接访问商店的：

1. 系统自动识别分销代码
2. 在订单处理过程中记录分销信息
3. 订单支付确认后，系统自动计算分销佣金
4. 佣金信息记录在系统中，并在适当时候支付给分销商

## 异常处理

### 支付超时

1. 如果用户在指定时间内（通常为24小时）未完成支付：
   - Issue 会被标记为 `payment-timeout`
   - 用户可以在 Issue 中请求重新生成支付信息

### 订单取消

1. 用户可以在支付前通过关闭 Issue 来取消订单
2. 已支付的订单如需取消，需要在 Issue 中与商家协商

### 退款流程

1. 如果需要退款，用户可以在 Issue 中提出申请
2. 商家审核后可以通过 USDT 将款项退回用户的钱包地址
3. 退款完成后，Issue 会被标记为 `refunded`

## 安全考虑

- 所有敏感信息（如收货地址、联系方式）都使用 RSA 加密
- 加密数据只能由持有私钥的商家解密
- 支付过程通过区块链进行，具有透明性和不可篡改性
- 用户可以通过 GitHub 的身份验证机制确保订单的真实性

## 总结

GitShop 的购物流程充分利用了 GitHub 平台的功能，创建了一个安全、透明且高效的电子商务体验。通过结合加密技术和区块链支付，确保了用户信息的安全和交易的可靠性。
